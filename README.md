一个加密视频dump工具（.js文件hook了arraybuffer，破解对象是央视网直播，其采用了帧加密，一个基于hls.js的视频播放器，解密会调用wasm，wasm会验证用户环境，
由于浏览器原生只支持fmp4，所以一定会有解密过后的明文数据出现在内存。
具体逻辑是在wasm交解密数据给mse的过程，
需要用arraybuffer传递，此时会触发js的下载，
下载到一堆解密后的fmp4碎片，视频和音频流是分开的，
前两个下载的是音频视频的moov+ftyp，之后是音频视频的moof+mdat，
java程序的作用是把下载下来的碎片区分开音频视频和ftyp，
自动按照pts顺序合并输出一个独立的视频文件和音频文件，之后可用ffmpeg合并输出两个文件为一个mp4，
此思对大多基于wasm严重加密的视频网站适用。

js使用方式：打开devtools，定位下ts，确定调用了浏览器mse，才能用，
在页面加载完成和视频播放之前任意位置断点，刷新，在页面停住的时候控制台粘贴js
之后浏览器要允许下载多个文件，静静等待视频播放完成之后运行java程序整理成两个mp4(标准扩展名应该是.m4s，懒得改了)，
再用ffmpeg合并成一个mp4
